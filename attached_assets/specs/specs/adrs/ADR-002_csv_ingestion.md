# ADR-002: CSV Ingestion for Invoice Data

**Date:** 15 March 2026
**Status:** Accepted

## Context

The canary project requires a mechanism to ingest invoice data from creditors. The process must support the "done criterion" of being able to "input arbitrary but valid sets of invoice data." The chosen mechanism needs to be simple, robust, and accessible to creditors who may use a variety of accounting systems.

## Decision

Invoice data will be ingested into the system via `.csv` files. Creditors can provide CSV files of any size, at any time. Each row in the CSV will represent a single invoice. The file must adhere to a standard format where the columns map to the invoice data model defined in ADR-001.

The expected CSV format is:

`creditor_name,debtor_name,amount_due`

The `invoice_id` will be generated by the system upon ingestion and will not be included in the uploaded file.

## Rationale

CSV is a universal, plain-text format that is easy for both humans and machines to read and generate. This choice allows creditors to export data from virtually any spreadsheet or accounting application, fulfilling the requirement to handle data from arbitrary sources without building complex, specific integrations. It directly supports the need to ingest "arbitrary but valid sets of invoice data" in batches of "arbitrary size."

This approach avoids vendor lock-in and keeps the initial implementation simple and focused on the core task of building the graph.

## Consequences

- A clear specification for the CSV format (column headers, order, and encoding) must be documented and communicated to users providing the data.
- The system will require a file upload interface for operators (to be addressed in Group 4: Operator UI).

- The system must be designed to process invoices in batches as they are received from the CSV files.

## Review Trigger

This decision should be revisited if the project requirements evolve to include real-time data ingestion via an API, or direct integration with specific accounting platforms (e.g., Xero, QuickBooks).
